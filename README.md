# 📘 Инструкция по использованию `teltonika_autoupdate.py`

---

## 📌 Назначение

Скрипт автоматизирует полный цикл настройки роутера **Teltonika RUT200**:

* прошивка через Recovery Mode
* восстановление конфигурации из backup
* смена LAN IP
* настройка WireGuard
* перезапуск и проверка

---

## 📂 1. Структура файлов

В одной директории должны находиться:

```text
project/
├── teltonika_for_argus.py
├── update.bin
└── backup.tar.gz
```

| Файл            | Назначение             |
| --------------- | ---------------------- |
| `update.bin`    | прошивка роутера       |
| `backup.tar.gz` | резервная конфигурация |

ВАЖНО!
Для использования бэкапа, нужен пароль который использовался для **входа в веб-морду на роутере с которого стянули бэкап!** 

---

## ⚙️ 2. Установка зависимостей

```bash
pip install -r requirements.txt
```


---

## 🔌 3. Подготовка роутера

1. Отключить питание
2. Зажать **RESET**
3. Включить питание (держать RESET 7 морганий светодиодов)
4. Дождаться Recovery Mode
5. Подключить ПК кабелем

Адрес Recovery:

```text
http://192.168.1.1
```

---

## ▶️ 4. Запуск

### Linux

```bash
sudo python3 teltonika_for_argus.py
```

### Windows

```powershell
py teltonika_for_argus.py
```

⚠️ Требуются права администратора

---

## ⌨️ 5. Ввод параметров

### Пример

```text
LAN IP:        172.26.15.152/29
WG IP:         172.26.10.133 (Тут автоматически применяет /26 маску, пока меняется на 723 строке кода)
Root password: наклейка на роутере либо admin01
Backup pass:   
Peer key:      QJ4Rxxxx...
PSK:           4fA9xxxx...
```

### Описание

| Параметр        | Описание               |
| --------------- | ---------------------- |
| LAN IP          | новый IP роутера       |
| WG IP           | адрес WireGuard        |
| Root password   | пароль после прошивки  |
| Backup password | пароль из backup       |
| Peer key        | публичный ключ сервера |
| PSK             | preshared key          |

---

## 🌐 6. Временные IP

Скрипт добавляет:

| Назначение    | Пример           |
| ------------- | ---------------- |
| Recovery      | 192.168.1.10/24  |
| Backup доступ | 172.26.15.154/29 |
| Новый LAN     | 172.26.15.164/29 |

Скрипт высчитывает +2 к адресам с /29 маской


Удаляются автоматически после завершения работы скрипта

## ❗ 9. Частые ошибки

### Нет файлов

```text
Файл не найден
```

→ Проверить директорию, должны быть в одной папке файлы
```text
update.bin
&
backup.tar.gz` 
```

---

### Нет paramiko

```text
ModuleNotFoundError
```

→ Установить зависимости (см. пункт 2)

---

### Нет доступа к 192.168.1.1

Причины:

* не Recovery Mode(выводит уведомление в терминал после ввода параметров)
* кабель не подключён

---

### SSH не поднялся

Причины:

* неверный пароль
* другой IP в backup
(см. отладочную информацию в терминале)

---

### WireGuard не работает

Проверить:

* ключи
* allowed_ips
* конфиг пира
тестилось на одном бэкапе, могут быть не точности(см. отладочную информацию в терминале)

---

## ⚠️ 10. Рекомендации

* сразу выписать пароль, и WG-ключи
* не выключать питание при прошивке
* не закрывать скрипт
* при сбое использовать Recovery Mode
